# 앱 사용자 경험 보장 가이드 (Anti-Dark Pattern)

사용자에게 신뢰받는 서비스를 만들기 위해, **고객의 예상을 벗어나는 설계(Dark Pattern)**는 반드시 지양해야 합니다. 아래는 사용자의 자율성을 존중하고 긍정적인 경험을 제공하기 위한 핵심 원칙입니다.

## 1. 진입 시점의 방해 요소 제거 (No Immediate Interruptions)

- **원칙**: 앱 실행 직후에는 사용자가 기대한 홈 화면이나 콘텐츠를 먼저 보여주어야 합니다.
- **지양해야 할 패턴**: 앱에 진입하자마자 전면 광고, 알림 동의 요청 바텀시트(Bottom Sheet), 프로모션 모달 등이 화면을 가로막는 경우.
- **권장**: 필요한 권한이나 마케팅 정보는 사용자가 해당 기능에 접근하거나 관심을 보일 때 문맥에 맞게 요청하세요.

## 2. 내비게이션 흐름 존중 (Respect Navigation Intent)

- **원칙**: 사용자의 '뒤로 가기' 혹은 '나가기' 의도를 존중해야 합니다.
- **지양해야 할 패턴**: 뒤로 가기(Back) 버튼을 눌렀을 때, 이전 화면으로 이동하는 대신 광고나 "정말 나가시겠습니까?" 같은 방어적인 바텀시트를 띄워 이탈을 막는 행위.
- **권장**: 사용자가 앱을 탐색하는 흐름을 인위적으로 끊지 말아야 합니다.

## 3. 명확한 탈출구 제공 (Provide Clear Exit Paths)

- **원칙**: 모든 팝업, 모달, 화면에는 닫기(Close), 취소(Cancel), 혹은 뒤로 갈 수 있는 명확한 수단이 있어야 합니다.
- **지양해야 할 패턴**: 긍정적인 액션(CTA) 버튼만 존재하고, 닫거나 거절할 수 있는 버튼이 없거나 숨겨져 있는 경우.
- **권장**: 사용자가 원하지 않는다면 언제든지 해당 흐름에서 벗어날 수 있는 '선택권'을 주어야 서비스에 대한 반감을 줄일 수 있습니다.

## 4. 예측 가능한 광고 노출 (Predictable Ad Placement)

- **원칙**: 광고는 사용자의 작업 흐름을 방해하지 않는 상식적인 위치와 시점에 노출되어야 합니다.
- **지양해야 할 패턴**: 메뉴를 클릭하거나 화면을 전환하는 등 일반적인 조작 중에 예고 없이 전면 광고가 튀어나오는 경우.
- **권장**: 보상형 광고처럼 사용자가 인지하고 선택한 시점에만 광고를 제공하여 사용 흐름의 맥락을 유지하세요.

## 5. 정직하고 명확한 버튼 (Clear & Honest CTAs)

- **원칙**: 버튼(CTA)의 라벨은 클릭 시 발생할 결과를 정확하게 설명해야 합니다.
- **지양해야 할 패턴**:
    - 버튼 내용이 모호하여 결과를 예측할 수 없는 경우.
    - 단순히 "확인", "좋아요"처럼 구체적이지 않은 텍스트로 유도하는 경우.
    - 버튼 주변에 과장되거나 혼란을 주는 보조 텍스트를 배치하는 경우.
- **권장**: "쿠폰 받기", "결제하기", "삭제하기" 등 구체적인 동사형을 사용하여 사용자의 불안감을 해소하세요.
