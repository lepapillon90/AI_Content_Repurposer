# 앱 개발 핵심 가이드 (Branding & UI/UX)

성공적인 앱 서비스를 위해 일관된 브랜드 경험과 사용자 친화적인 UI/UX를 제공하는 것이 중요합니다. 아래 가이드는 앱 개발 시 반드시 고려해야 할 핵심 요소들입니다.

## 1. 브랜드 아이덴티티 (Brand Identity)

사용자가 앱을 처음 접했을 때, 그리고 사용하는 내내 브랜드의 정체성을 명확하게 인지할 수 있어야 합니다.

### 브랜드 로고 (Logo)
- **가독성**: 작은 아이콘부터 큰 스플래시 화면까지 깨짐 없이 선명해야 합니다 (SVG 권장).
- **배경 조화**: 라이트/다크 모드 등 다양한 배경에서도 로고가 잘 보이도록 대비를 고려하거나, 전용 배경색을 포함한 아이콘을 사용합니다.
- **단순함**: 복잡한 디테일보다는 심볼 위주의 단순하고 명확한 형태가 기억에 오래 남습니다.

### 브랜드 네임 (App Name)
- **간결성**: 앱 이름은 직관적이고 기억하기 쉬워야 합니다.
- **일관성**: 앱 스토어, 홈 화면, 앱 내 헤더 등 모든 곳에서 동일한 표기를 사용합니다.

### 브랜드 컬러 (Color System)
- **Primary Color**: 브랜드를 대표하는 메인 컬러를 정의하고, 주요 액션 버튼(CTA)이나 강조 요소에 일관되게 사용합니다.
- **Secondary Color**: 메인 컬러를 보조하거나, 상태(성공, 에러, 경고 등)를 나타내는 기능적 컬러를 정의합니다.
- **접근성**: 텍스트와 배경 간의 명도 대비(Contrast Ratio)를 충분히 확보하여 가독성을 높입니다.

## 2. 내비게이션 바 (Navigation Bar)

화면 상단에 위치하여 현재 위치를 알려주고, 뒤로 가기나 중요 기능에 접근하는 영역입니다.

- **명확한 타이틀**: 현재 화면이 무엇인지 직관적으로 알 수 있는 타이틀을 제공합니다.
- **일관된 위치**: 뒤로 가기 버튼, 메뉴 버튼 등의 위치를 모든 화면에서 통일합니다.
- **플랫폼 표준 준수**: iOS(Human Interface Guidelines)와 Android(Material Design)의 각 플랫폼별 내비게이션 표준을 존중하되, 브랜드 정체성을 해치지 않는 선에서 커스터마이징합니다.

## 3. 화면 전환 및 진입 (Transitions & Entry)

앱 간 이동이나 화면 간 전환은 사용자에게 "끊김 없는(Seamless)" 경험을 제공해야 합니다.

- **스플래시 스크린**: 앱 실행 시 짧게 노출되는 브랜딩 화면으로, 로딩 시간을 체감상 줄여주는 역할을 합니다.
- **자연스러운 애니메이션**: 화면 이동 시 뚝 끊기는 느낌보다, 슬라이드나 페이드 효과를 적절히 사용하여 흐름을 연결합니다.
- **딥링크 지원**: 푸시 알림이나 외부 링크를 통해 진입할 때, 메인 화면이 아닌 특정 콘텐츠 화면으로 바로 연결되는 경험을 제공해야 합니다.

## 4. 탭 바 (Tab Bar / Bottom Navigation)

앱의 최상위 레벨 메뉴를 이동하는 데 사용되는 가장 중요한 내비게이션 요소입니다.

- **단순한 구조**: 탭 메뉴는 **최소 3개에서 최대 5개**를 권장합니다. 5개를 넘어가면 터치 영역이 좁아지고 인지 부하가 커집니다.
- **직관적인 아이콘 & 라벨**: 아이콘만으로 의미가 불분명할 경우 텍스트 라벨을 함께 표기합니다.
- **현재 위치 표시**: 사용자가 현재 어떤 탭에 있는지 색상이나 아이콘 스타일(Filled vs Outlined)로 명확히 구분해 줍니다.
- **상태 유지**: 탭 간 이동 시, 이전 탭에서 보던 스크롤 위치나 입력 상태가 유지되도록 구현하는 것이 좋습니다.
