# 앱 UX 라이팅 가이드 (UX Writing)

사용자에게 쉽고 친근하게 다가가는 글쓰기 원칙입니다. 기술적인 용어 대신 일상적인 대화체를 사용하여 사용자의 이해를 돕고 행동을 명확하게 유도합니다.

## 1. 친근한 대화체 사용 (Consistent Tone)

- **원칙**: 딱딱한 문어체나 명령조 대신, 부드러운 구어체인 **'해요체'**를 일관되게 사용합니다.
- **예시**:
    - `로그인이 필요합니다.` (X) -> `로그인해주세요.` (O)
    - `전송 완료.` (X) -> `전송했어요.` (O)

## 2. 능동적인 표현 (Active Voice)

주체를 명확히 하고, 사용자가 주도적으로 행동하는 느낌을 줍니다.

### '됐어요' 피하기
무책임하게 느껴질 수 있는 피동형 표현을 지양합니다.
- **Don't**: `등록이 완료됐어요.`
- **Do**: `등록을 완료했어요.`

### 문장 간결하게 만들기 ('~었' 빼기)
불필요한 과거형 어미를 줄여 문장을 짧고 경쾌하게 만듭니다.
- **Don't**: `송금을 보냈어요.` (이미 행동이 끝난 후 결과 화면 등에서)
- **Do**: `송금 완료` 또는 `송금을 보냈어요` -> `송금했어요` (상황에 따라 더 자연스러운 쪽 선택, 보통은 현재 결과 중심)
- *Toss 가이드 참조: 기계적인 과거형보다는 현재 상태나 완료된 사실을 담백하게 전달.*

## 3. 긍정적인 언어 (Positive Language)

'안 된다'는 말보다 '어떻게 하면 되는지'를 알려줍니다.

### '없어요' 대신 '있어요'
- **Don't**: `지금은 이 기능을 쓸 수 없어요.`
- **Do**: `이 기능은 내일 아침 9시부터 쓸 수 있어요.`

### 에러 메시지는 해결책 중심으로
단순히 에러 사실만 통보하지 말고, 다음 행동을 안내합니다.
- **Don't**: `네트워크 연결 실패`
- **Do**: `인터넷 연결을 확인하고 다시 시도해주세요.`

## 4. 쉽고 편안한 경어 (Casual Honorifics)

과도한 존칭은 오히려 거리감을 만들 수 있습니다. 꼭 필요한 경우에만 정중하게 표현합니다.

### 불필요한 '시' 빼기
시스템 메시지나 단순 안내에서는 과한 높임말을 줄입니다.
- **Don't**: `비밀번호를 입력하시면, 다음 단계로 이동합니다.`
- **Do**: `비밀번호를 입력하면, 다음 단계로 이동해요.`

### 쉬운 단어 선택
- `계시다` -> `있다`
- `여쭈다` -> `물어보다` / `확인하다`
- `~께` -> `~에게`

*단, 사용자의 양해를 구하거나 민감한 정보를 다룰 때는 정중한 표현(경어)을 사용합니다.*

## 5. 명사 나열 피하기 (Avoid Noun Clusters)

한자어 명사나 딱딱한 단어의 나열 대신, 풀어서 설명합니다.

### 한자어 풀어쓰기
- **Don't**: `본인 인증 완료` (기계적)
- **Do**: `본인 인증을 마쳤어요` (자연스러움)

### 기능 명칭 반복 줄이기
기능 이름 뒤에 '~하기'를 붙여 동명사 꼴로 만드는 것을 피하고, 동사로 마무리합니다.
- **Don't**: `계좌 개설하기` (버튼이나 타이틀에서)
- **Do**: `계좌 만들기`
