# 앱 그래픽/리소스 가이드 (Graphic Design)

그래픽은 단순한 장식이 아니라, 사용자의 이해를 돕고 직관적인 전달을 위한 핵심 수단입니다. 깔끔하고 일관성 있는 비주얼을 제공하기 위해 아래 가이드를 참고하세요.

## 1. 문맥에 맞는 그래픽 사용 (Contextual Graphics)

- **원칙**: 그래픽은 정보를 더 빠르고 쉽게 전달해야 합니다.
- **가이드**:
    - 장식적인 목적보다는 **기능적 목적**에 집중하세요.
    - 텍스트만으로 전달하기 어려운 상태나 감정을 시각적으로 보완합니다.
    - 상황과 무관한 '예쁜' 아이콘보다는 상황을 '설명하는' 아이콘을 선택하세요.

## 2. 적절한 밀도와 크기 (Density & Size)

- **원칙**: 정보의 위계에 따라 그래픽의 복잡도와 크기를 조절합니다.
- **가이드**:
    - **작은 크기 (아이콘)**: 단순하고 명료한 라인과 형태를 사용합니다. (24~40px 권장)
    - **큰 크기 (일러스트/배너)**: 좀 더 디테일한 표현이 가능하지만, 너무 복잡하지 않게 유지합니다.
    - **균형**: 한 화면에 비슷한 비중의 그래픽이 여러 개 있으면 시선이 분산됩니다. 핵심 그래픽 1개에 집중하고 나머지는 보조적으로 활용하세요.

## 3. 다크 모드 대응 (Accessibility)

- **원칙**: 모든 사용 환경에서 그래픽이 명확하게 보여야 합니다.
- **가이드**:
    - **Dark/Light 모두 대응**: 배경색이 바뀌어도 아이콘이나 이미지가 묻히지 않도록 중간 명도를 활용하거나, 모드별 에셋을 분리하여 제공합니다.
    - **투명도 활용**: 배경이 있는 이미지보다는 투명(PNG/SVG) 배경을 사용하여 다양한 테마에 녹아들게 합니다.

## 4. 시각적 방해 요소 제거 (Avoid Distractions)

- **원칙**: 콘텐츠가 주인공이어야 합니다. 그래픽이 콘텐츠를 가리거나 방해해서는 안 됩니다.
- **지양해야 할 것**:
    - 과도한 파티클, 의미 없는 그라데이션, 지나치게 화려한 이펙트.
    - 텍스트나 버튼을 가리는 배치.
    - 부정적이거나 호소하는(애원하는) 감정 표현의 그래픽.

## 5. 일관된 스타일 (Consistency)

- **원칙**: 앱 전체에서 하나의 브랜드가 느껴지도록 통일된 화풍을 유지합니다.
- **가이드**:
    - **톤앤매너**: 선의 굵기, 코너의 둥글기(Radius), 사용하는 컬러 팔레트를 통일합니다.
    - **스타일**: 갑자기 만화 같은 그림이 나왔다가 실사가 나오는 등 스타일이 뒤섞이지 않도록 주의합니다.
    - **단순 명료함**: 모바일 환경에서는 단순하고 깨끗한 스타일이 정보 전달에 가장 효과적입니다.
