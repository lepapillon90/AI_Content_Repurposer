# Project Rules & Guidelines

## 0. Communication & Language Rules
1.  **Language**: Always respond in **Korean (한국어)**.
2.  **Code Comments**: 주석도 반드시 **한국어**로 작성할 것.
3.  **Explanation**: 기술적인 용어는 영어를 병기하되(ex: 변수(Variable)), 설명은 쉽게 한국어로 풀어서 할 것.

## 1. Code Reusability & File Structure
1.  **File Separation**: 코드가 길어지면 파일을 분리하여 **재사용성(Reusability)**을 높일 것. (예: `utils.js`, `constants.js`)
2.  **Component Extraction**: 반복되는 UI나 로직은 별도의 함수나 컴포넌트로 분리할 것.
3.  **Single Responsibility**: 하나의 파일/함수는 하나의 역할만 담당하도록 **단일 책임 원칙(SRP)**을 준수할 것.

## 2. Tech Stack & performance Rules (RepurposeAI Specific)
1.  **Vanilla Stack**: 빌드 도구 없이 `index.html`에서 바로 실행 가능한 구조를 유지할 것.
2.  **Library Usage**: 외부 라이브러리는 `npm install` 대신 **CDN** 스크립트 태그로 불러올 것. (Tailwind, Lucide Icons 등)
3.  **Responsiveness**: 모든 UI는 **모바일 우선(Mobile First)**으로 설계하고 데스크톱까지 대응할 것.

## 3. UI/UX Design Standards
1.  **Branding**: 메인 컬러 **Deep Blue (#1E40AF)**를 버튼 및 강조 요소에 일관되게 사용할 것.
2.  **Feedback**: 사용자의 모든 액션(클릭, 복사, 생성)에는 즉각적인 시각적 피드백(로딩 스피너, 토스트 메시지)을 제공할 것.
3.  **Typho**: 폰트는 `Inter` 또는 `Pretendard`를 사용하여 가독성을 확보할 것.

## 4. Naming Conventions
1.  **Variables/Functions**: `camelCase` 사용 (예: `generateContent`, `userInput`).
2.  **Constants**: `UPPER_SNAKE_CASE` 사용 (예: `MAX_CHAR_COUNT`).
3.  **CSS Classes**: Tailwind 유틸리티 클래스 위주로 사용하되, 커스텀 CSS가 필요한 경우 `kebab-case` 사용.
4.  **Files**: `kebab-case` 또는 `snake_case` 사용 (예: `user_flow.md`, `style.css`).
