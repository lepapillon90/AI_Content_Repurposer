# 🔄 03_USER_FLOW.md

> **목표**: 마찰 최소화. 사용자는 가능한 적은 클릭으로 "아이디어"에서 "재가공된 콘텐츠"로 이동해야 합니다.

---

## 메인 플로우: "리퍼포징 루프"

### 1단계: 랜딩
*   **사용자 행동**: 웹 앱을 엽니다.
*   **시스템**: 깔끔한 인터페이스를 표시합니다. 입력 영역이 포커스되거나 눈에 띕니다.
*   **비주얼**: 타이틀 "RepurposeAI"와 "Turn one content into many".

### 2단계: 입력
*   **사용자 행동**: 블로그 게시물 텍스트를 왼쪽 텍스트 영역에 붙여넣습니다.
*   **시스템**: 글자 수를 실시간으로 업데이트합니다.

### 3단계: 선택
*   **사용자 행동**: "플랫폼" 드롭다운을 클릭합니다 (기본값: Twitter Thread).
*   **사용자 행동**: 타겟을 선택합니다 (예: "LinkedIn Post").
*   **시스템**: "Generate" 버튼 라벨을 업데이트합니다 (예: "Generate LinkedIn Post").

### 4단계: 생성
*   **사용자 행동**: "Generate" 버튼을 클릭합니다.
*   **시스템**:
    1.  버튼 상태가 "Generating..."으로 변경됩니다 (스피너 아이콘).
    2.  AI 처리를 시뮬레이션합니다 (1.5초 지연).
    3.  오른쪽 출력 카드에 포맷팅된 텍스트를 채웁니다.
    4.  콘텐츠 분석 데이터(Viral Score, Pillars, Keywords)를 시연합니다.
    5.  버튼이 정상 상태로 돌아옵니다.

### 4.5단계: 분석 및 조정
*   **사용자 행동**: 콘텐츠 영역 왼체의 Viral Score 점수를 확인합니다.
*   **사용자 행동**: "Hook", "Value" 등 개별 뱃지를 클릭합니다.
*   **시스템**: 해당 지표에 대한 AI의 구체적인 코멘트와 점수를 담은 모달을 표시합니다.
*   **사용자 행동**: 피드백을 바탕으로 에디터에서 콘텐츠를 수정합니다.

### 5단계: 액션
*   **사용자 행동**: 콘텐츠를 검토합니다.
*   **사용자 행동**: "Copy" 아이콘 버튼을 클릭합니다.
*   **시스템**:
    1.  텍스트를 시스템 클립보드에 복사합니다.
    2.  일시적인 "Copied!" 토스트/툴팁을 표시합니다.
    3.  사용자는 소셜 미디어에 붙여넣을 준비가 되었습니다.

### 6단계: 히스토리 활용
*   **사용자 행동**: 사이드바의 "History" 또는 "Dashboard"를 클릭합니다.
*   **사용자 행동**: 과거에 생성한 콘텐츠를 목록에서 선택합니다.
*   **시스템**: 에디터 위젯을 열고, 당시의 생성 콘텐츠와 분석 데이터(Viral Score 등)를 모두 복원하여 보여줍니다.
