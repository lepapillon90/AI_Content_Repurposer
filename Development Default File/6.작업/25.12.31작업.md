# 2025.12.31 작업 내역

오늘 진행된 주요 작업 내용입니다. SPA 구조로의 전환, 컴포넌트 모듈화, 그리고 주요 UI/UX 버그 수정이 포함되었습니다.

## 1. SPA (Single Page Application) 구조 전환
- **통합**: `dashboard.html`의 기능을 `index.html`로 완전히 통합했습니다.
- **라우팅**: 해시 기반 라우팅(`#home`, `#dashboard`, `#auth/signin` 등)을 도입하여 페이지 새로고침 없이 뷰가 전환됩니다.
- **히스토리 관리**: 브라우저 히스토리와 연동되어 뒤로가기/앞으로가기가 정상 작동합니다.

## 2. 컴포넌트 모듈화 (`js/components.js`)
- **재사용성**: 헤더(Header)와 사이드바(Sidebar)를 자바스크립트 컴포넌트로 분리했습니다.
- **동적 렌더링**: 페이지 로드 시 또는 상태 변경(로그인/로그아웃) 시 UI가 동적으로 업데이트됩니다.
- **전역 접근**: `window.Components`를 통해 앱 어디서든 컴포넌트 렌더링 함수에 접근할 수 있습니다.

## 3. UI/UX 개선 및 버그 수정
- **사이드바 네비게이션**: 
    - 모바일 환경에서 햄버거 메뉴를 통한 사이드바 열기/닫기 기능을 구현했습니다.
    - 활성 메뉴(Active State)가 현재 뷰에 맞춰 자동으로 하이라이트됩니다.
    - `transition` 속성 조정으로 페이지 로드 시 발생하는 레이아웃 깜빡임(Flash) 현상을 해결했습니다.
- **다크 모드**: 
    - 동적으로 생성되는 헤더 내의 다크 모드 토글 버튼이 작동하지 않는 문제를 **이벤트 위임(Event Delegation)** 방식으로 해결했습니다.
- **푸터(Footer)**: 
    - 사용자 피드백을 반영하여 초기 고정(Fixed) 시도 후, 최종적으로 제거하여 화면을 더 넓고 깔끔하게 사용할 수 있도록 했습니다.
- **헤더 정리**: 
    - 사이드바와 기능이 중복되는 헤더의 'Dashboard' 버튼을 제거했습니다.

## 4. 주요 버그 수정
- **초기화 오류(TDZ Issue)**: `app.js`에서 변수가 초기화되기 전에 함수가 실행되어 버튼이 먹통이 되던 문제를 해결했습니다 (`DOMContentLoaded` 실행 순서 조정).
- **이벤트 리스너 소실**: HTML을 덮어쓰면서(innerHTML) 사라지던 이벤트 리스너들을 `document` 레벨에서 위임 처리하여 안정성을 확보했습니다.
